## 🚀 Panduan Deployment ke VPS (Next Steps)
Karena di VPS target sudah ada web lain (milik teman) yang menggunakan Nginx, skenario deployment-nya adalah **Reverse Proxy Integration**:
1. **Pull Kode:** Jalankan `git pull origin release/v1.0-ready` (atau branch `main` jika sudah di-merge) di VPS.
2. **Konfigurasi Lingkungan:** Buat `.env.prod` berdasarkan `.env.prod.example`.
3. **Modifikasi Port:** 
   - Di file `docker-compose.prod.yml`, berikan akses port keluar untuk web (contoh: `4321:4321`) dan directus (`8055:8055`).
   - Hapus atau *comment* konfigurasi container `voda-nginx-prod` dan `voda-certbot-prod` di file tersebut agar tidak bentrok memperebutkan port 80/443 dengan Nginx teman Anda.
4. **Numpang Nginx Utama:** Minta tolong pemilik VPS/Nginx saat ini untuk menambahkan blok rute:
   - `vodatrip.id` diarahkan ke localhost port `4321`.
   - `api.vodatrip.id` diarahkan ke localhost port `8055`.
   - Biarkan Nginx utama yang mengatur SSL/HTTPS (Certbot).
5. **Jalankan:** Eksekusi `docker compose -f docker-compose.prod.yml up -d --build`.
